# Problem And Goal

## 문제
사용자는 WorkProof 화면에서 "현재 위치 확인"이 느리다고 느끼고 있다.

현재 구조상 위치 조회 지연은 한 가지 원인이 아니라 아래가 합쳐질 수 있다.

- 단말 GPS provider 응답 지연
- provider 순차 재시도와 timeout 누적
- last known fallback 허용 기준
- 화면 진입 자동 조회 + 수동 조회 + 출퇴근 전 재조회 중복

그래서 이번 트랙은 "느리다"를 바로 고치는 대신, 먼저 "어디서 느린가"를 구간별로 분리해서 확인한다.

## 목표
이번 트랙의 목표는 아래 네 가지다.

1. 현재 위치 조회 지연을 숫자로 확인한다.
2. 조회 시간을 provider/path 단위로 분해해 가장 큰 병목을 찾는다.
3. 사용자 체감 지연을 줄이는 개선을 작은 리스크로 적용한다.
4. before/after 결과를 문서로 남겨 재현 가능하게 만든다.

## 성공 기준
- 최소 한 개 핵심 지표에서 명확한 before/after 개선을 만든다.
- 위치 정확도/반경 판정 안정성을 깨지 않는다.
- 출퇴근 제출 경로의 validation/security 계약을 깨지 않는다.

## 지금 당장 답해야 하는 질문
- "느리다"는 화면 진입 시점인지, 버튼(현재 위치 핀 이동) 시점인지, 출퇴근 제출 시점인지
- 조회 지연의 본체가 provider 대기인지, fallback 판단인지, 중복 호출인지
- 단말/환경(GPS on/off, 실내/실외)에 따라 지연 패턴이 얼마나 달라지는지
