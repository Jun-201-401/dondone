# WorkProof Statement Document Purpose and Sections

## 목적
- `WORKPROOF_STATEMENT`는 선택한 기간의 출퇴근 기록과 변경 이력을 사람이 읽기 쉬운 형태로 정리한 PDF 문서다.
- 이 문서는 세 가지 용도를 동시에 만족해야 한다.
  - 근로자가 자신의 근무 기록을 기간 기준으로 확인할 수 있어야 한다.
  - 고용주가 근무 기록과 수정 이력을 빠르게 검토할 수 있어야 한다.
  - 임금체불 진정/상담 시 첨부 문서로 제출해도 문맥이 이해되어야 한다.

## 비목적
- 급여 계산 결과를 확정하는 문서가 아니다.
- 회사 내부 정산 시스템용 상세 원장 문서가 아니다.
- 첨부 원본 파일이나 위치 원본 로그를 모두 포함하는 포렌식 보고서가 아니다.
- `proof pack`이라는 내부 용어를 전면에 드러내는 문서가 아니다.

## 대상 독자
- 1차 독자: 근로자 본인
- 2차 독자: 고용주 또는 현장 관리자
- 3차 독자: 노무 상담 담당자, 진정서 검토자, 분쟁 조정 과정의 제3자

## 문서 제목 원칙
- 사용자에게 보이는 기본 제목은 `근무 기록 문서`로 한다.
- 영문 템플릿 식별자는 `WorkProof Statement`를 유지할 수 있다.
- 문서 내부에서는 `Proof Pack` 같은 내부 기능 이름을 기본 제목으로 사용하지 않는다.

## 문서 성격
- 기간 기반 문서다.
- 근무지 기준 문서다.
- 읽기 우선 문서다.
- 증빙 보조 문서다.

## 문서가 반드시 전달해야 하는 질문
- 누구의 기록인가
- 어느 근무지 기록인가
- 어느 기간 기록인가
- 실제 출근/퇴근 기록은 어떻게 되었는가
- 수정된 기록이 있는가
- 검토가 필요한 기록이 있는가

## v1 포함 섹션
1. 문서 헤더
   - 문서명
   - 문서번호
   - 생성 시각
   - 대상 기간

2. 근로자 정보
   - 이름
   - 이메일
   - 사용자 식별 정보 최소값

3. 근무지 정보
   - 근무지명
   - 주소
   - 위치 라벨

4. 기간 요약
   - 총 기록 수
   - 반영 완료 건수
   - 검토 필요 건수
   - 수정 기록 건수
   - 첨부 수
   - 총 근무시간

5. 출퇴근 기록 표
   - 날짜
   - 출근 시각
   - 퇴근 시각
   - 근무시간
   - 상태
   - 수정 여부
   - 첨부 수
   - 비고 또는 메모

6. 수정 이력 섹션
   - 수정 시각
   - 대상 기록 식별자
   - 변경 전 출근/퇴근
   - 변경 후 출근/퇴근
   - 수정 사유
   - 메모 변경 내용

7. 안내 문구 섹션
   - `검토 필요` 상태 의미
   - 반경 밖 퇴근 기록 안내
   - 첨부 수와 첨부 원본 미포함 안내

## v1 제외 섹션
- 급여 차액 설명
- 급여 검증 결과
- Claim Kit/진정서 본문
- 첨부 파일 원본 이미지 삽입
- 지도 이미지
- 블록체인/송금 정보
- QR 검증 링크
- 내부 디버그 정보

## 기록 표 컬럼 원칙
- 컬럼 수는 읽기성을 해치지 않는 선에서 최소화한다.
- 위치 원본 좌표는 표에 직접 노출하지 않는다.
- 상태는 내부 enum명이 아니라 사람이 읽는 라벨로 보여준다.
- 메모와 수정 사유는 너무 긴 경우 줄바꿈을 허용한다.

## 수정 이력 원칙
- 수정 이력이 없으면 섹션은 숨기거나 `수정 이력 없음` 수준으로 단순화한다.
- 수정 이력은 "무엇이 바뀌었는가"를 보여주는 데 집중한다.
- 내부 audit 상세 필드를 모두 노출하지 않는다.

## 안내 문구 원칙
- 방어적 법률 문구보다 읽기 쉬운 설명 문구를 우선한다.
- 판단을 강요하는 문구보다 상태 설명에 집중한다.
- `이 문서만으로 법적 결론이 확정된다`는 식의 표현은 사용하지 않는다.

## 스냅샷 재설계 기준
- 템플릿이 계산하지 않도록 표시용 라벨과 문자열은 snapshot에서 제공한다.
- 문서 목적과 직접 관련 없는 필드는 제거한다.
- 문서 제목, 기간 라벨, 상태 라벨, 총 근무시간 텍스트는 snapshot에서 완성한다.
- 계약 정보는 v1에서 필수가 아니므로, 템플릿 본문 핵심 섹션에서 우선순위를 낮춘다.

## 템플릿 재설계 기준
- A4 세로 기준으로 읽기 쉬운 단일 문서 구조를 유지한다.
- 헤더 -> 요약 -> 기록 표 -> 수정 이력 -> 안내 문구 순서를 유지한다.
- `근로자 확인용 + 제3자 검토용` 문서라는 점을 고려해 과도한 브랜드 장식은 줄인다.
- 출력 시 표 줄바꿈, 페이지 나눔, 긴 메모 처리 안정성을 우선한다.
